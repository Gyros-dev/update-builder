# Конструктор email-рассылки

Статический проект для GitHub Pages. Сервер и база данных не нужны.

## Возможности

- добавление любого количества обновлений;
- необязательный скриншот для каждого пункта;
- изменение порядка пунктов;
- автоматический предпросмотр;
- автосохранение в браузере;
- сохранение и загрузка черновика JSON;
- экспорт автономного HTML со встроенными логотипом и скриншотами.

## Публикация на GitHub Pages

1. Создайте новый репозиторий на GitHub.
2. Загрузите в него все файлы этой папки.
3. Откройте **Settings → Pages**.
4. В разделе **Build and deployment** выберите **Deploy from a branch**.
5. Выберите ветку `main`, папку `/ (root)` и нажмите **Save**.

## Как пользоваться

1. Откройте опубликованный сайт.
2. Заполните заголовок и описание релиза.
3. Добавьте обновления. Скриншот можно не добавлять.
4. Нажмите **Скачать HTML для отправки**.
5. Откройте скачанный файл в браузере.
6. Выделите содержимое письма (`Cmd+A` / `Ctrl+A`), скопируйте и вставьте в редактор письма.

> Важно: некоторые почтовые сервисы могут изменять HTML при вставке. Перед массовой отправкой сделайте тестовую отправку себе в Gmail и Outlook.
